L01: Z05

Treść:

Ułóż algorytm, który dla zadanego acyklicznego grafu skierowanego G znajduje długość najdłuższej drogi w G. Następnie zmodyfikuj swój algorytm tak, by wypisywał drogę o największej daugości (jeśli jest kilka takich dróg, to Twój algorytm powinien wypisać dowolną z nich).

Rozwiązanie:

Weżmy przykładowy graf G, spróbujmy ustalić długość jego najdłuższej ścieżki z wieszchołka A do wierzchołka E. Niech distance(v) oznacza najdłuższą ścieżkę z wierzchołka v do wierzchołka o stopniu wejściowym równym 0 (czyli najdłuższą ścieżkę kończącą się w v) - w tym przypadku do wierzchołka A. Widzimy, że z wierzchołka E do wierzchołka A możemy się dostać jedynie poprzez jego poprzedników (tj. wierzchołki D oraz B). Oznacza to, że aby otrzymać nadłuższą ścieżkę z A do E musimy znaleźć długość najdłuższej ścieżki z A do jego poprzedników(B i D), wybrać dłuższą z nich i dodać 1. Zatem możemy wyznaczyć tę długość za pomocą wzoru:

distance(E) = max(distance(B), distance(D)) + 1

Natomiast kolejne długości możemy wyznaczyć rekurencyjnie:

distance(D) = max(distance(A), distance(C)) + 1
distance(B) = max(distance(A)) + 1

I tak dalej aż do wyznaczenia długości. Widzimy, że rozwiązaniem dla powyższego przykładu jest ścieżka A->B->C->D->E.

Graf G

Graf G w reprezentacji liniowej (sortowanie topologiczne)

Przyjmujemy, że dla wierzchołka v o stopniu wejściowym 0 - dist(v) = 0. Wiemy z własności grafu skierowanego acyklicznego, że istnieje w nim co najmniej jeden wierzchołek o stopniu wejściowym równym 0, który występuje na początku liniowej reprezentacji.
Chcemy wyznaczyć najdłuższą drogę w oszczędny sposób, spójrzmy na rysunek przedstawiający graf G w reprezentacji liniowej. Jeśli rozpoczniemy szukanie najdłuższej drogi w kolejności danej przez tę reprezentację to dla każdego kolejnego wierzchołka będziemy znali już najdłuższe drogi dla jego poprzedników co usprawni nasz algorytm. Jego idea jest taka, że dla każdego kolejnego wierzchołka w liniowej reprezentacji będziemy obliczać najdłuższe dla niego długości korzystając z wcześniej już obliczonych długości jego poprzedników. Długości te obliczać będziemy z podanej wyżej zależności rekurencyjnej, którą wykorzystujemy w funkcji maxDistance().

Algorytm:

Dane:
predecessors[V] - graf G dany w postaci listy sąsiedztwa takiej, że jeśli wierzchołek u należy do predecessor[v] to istnieje krawędź (u, v)
topological_order[V] - lista wierzchołków grafu G w liniowej reprezentacji - O(E + V)
distance[V] = początkowo pusta lista najdłuższych odległości kończońcych się w danych wierzchołkach
V - liczba wierzchołków

longestPath() {
    var maxDistance = 0     // zmienna, w której trzymamy długość najdłuższej aktualnie ścieżki, wartość końcowa będzie równa długości najdłuższej ścieżki w grafie
    var endOfLongestPath    // chcemy wiedzieć, na którym wierzchołku kończy się najdłuższa ścieżka
    
    for vertex in topological_order {
        distance[vertex] = maxDistance(vertex, dist[])
        if distance[vertex] > maxDistance {
            maxDistance = dist[vertex]
            endOfLongestPath = vertex
        }
    }
    
    var path = getPath(endOfLongestPath, [endOfLongestPath])    // lista zawierająca naszą ścieżkę
    reverse(path)    // zauważamy, że dodawaliśmy wierzchołki od końca, zatem musimy odwrócić kolejność
    for vertex in path {
        print(vertex)
    }
}

maxDistance(v) {
    if predecessors[v].count == 0 {    // zgodnie z założeniami jeśli stopień wejściowy wierzchołka v jest równy zeru to ustawiamy dist[v] = 0
        return 0
    }
    var maxDistance = 0
    for predecessor in predecessors[v] {
        if distance[predecessor] > maxDistance {
            maxDistance = distance[predecessor]    // szukamy maksymalnej długości wśród poprzedników
        }
    }
    return maxDistance + 1
}

getPath(v, path[]) {
    var maxDistance = 0
    var previousVertex         // wierzchołek poprzednika v
    for neighbour in neighbours[v] {
        if distance[neighbour] > maxDistance {
            maxDistance = distance[neighbour]
            previousVertex = neighbour
        }
    }
    path += previousVertex
    if maxDistance == 0 {      // doszliśmy do początku ścieżki
        return path
    } else {
        getPath(previousVertex, path)
    }
}

Złożoność:

Złożoność czasowa:

  • wykonanie sortowania topologicznego wykona się w czasie O(V + E)
  • w longestPath() przechodzimy przez każdy wierzchołek grafu wywołując na nim maxDistance(), który porusza się po wszystkich wierzchołkach, które do niego trafiają. Zatem poruszamy się po każdym wierzchołku i każdej krawędzi zatem wykona się to w czasie O(V + E).
  • getPath(v) porusza się po poprzednikach v, których jest maksymalnie v - 1
  • zatem ogólna złożoność wynosi O(V + E)

Złożoność pamięciowa:

  • graf podany w postaci listy sąsiedztwa ma złożoność pamięciową równą O(V + E)
  • topological_order[] oraz distance[] mają złożoność O(V)
  • path[] ma złożoność co najwyżej O(V - 1)
  • reszta zmiennych ma złożoność O(1)
  • zatem ogólna złożoność wynosi O(V + E)
