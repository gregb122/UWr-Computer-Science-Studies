L01: Z05

Tre:

U algorytm, ktry dla zadanego acyklicznego grafu skierowanego G znajduje dugo najduszej drogi w G. Nastpnie zmodyfikuj swj algorytm tak, by wypisywa drog o najwikszej daugoci (jeli jest kilka takich drg, to Twj algorytm powinien wypisa dowoln z nich).

Rozwizanie:

Wemy przykadowy graf G, sprbujmy ustali dugo jego najduszej cieki z wieszchoka A do wierzchoka E. Niech distance(v) oznacza najdusz ciek z wierzchoka v do wierzchoka o stopniu wejciowym rwnym 0 (czyli najdusz ciek koczc si w v) - w tym przypadku do wierzchoka A. Widzimy, e z wierzchoka E do wierzchoka A moemy si dosta jedynie poprzez jego poprzednikw (tj. wierzchoki D oraz B). Oznacza to, e aby otrzyma nadusz ciek z A do E musimy znale dugo najduszej cieki z A do jego poprzednikw(B i D), wybra dusz z nich i doda 1. Zatem moemy wyznaczy t dugo za pomoc wzoru:

distance(E) = max(distance(B), distance(D)) + 1

Natomiast kolejne dugoci moemy wyznaczy rekurencyjnie:

distance(D) = max(distance(A), distance(C)) + 1
distance(B) = max(distance(A)) + 1

I tak dalej a do wyznaczenia dugoci. Widzimy, e rozwizaniem dla powyszego przykadu jest cieka A->B->C->D->E.

Graf G

Graf G w reprezentacji liniowej (sortowanie topologiczne)

Przyjmujemy, e dla wierzchoka v o stopniu wejciowym 0 - dist(v) = 0. Wiemy z wasnoci grafu skierowanego acyklicznego, e istnieje w nim co najmniej jeden wierzchoek o stopniu wejciowym rwnym 0, ktry wystpuje na pocztku liniowej reprezentacji.
Chcemy wyznaczy najdusz drog w oszczdny sposb, spjrzmy na rysunek przedstawiajcy graf G w reprezentacji liniowej. Jeli rozpoczniemy szukanie najduszej drogi w kolejnoci danej przez t reprezentacj to dla kadego kolejnego wierzchoka bdziemy znali ju najdusze drogi dla jego poprzednikw co usprawni nasz algorytm. Jego idea jest taka, e dla kadego kolejnego wierzchoka w liniowej reprezentacji bdziemy oblicza najdusze dla niego dugoci korzystajc z wczeniej ju obliczonych dugoci jego poprzednikw. Dugoci te oblicza bdziemy z podanej wyej zalenoci rekurencyjnej, ktr wykorzystujemy w funkcji maxDistance().

Algorytm:

Dane:
predecessors[V] - graf G dany w postaci listy ssiedztwa takiej, e jeli wierzchoek u naley do predecessor[v] to istnieje krawd (u, v)
topological_order[V] - lista wierzchokw grafu G w liniowej reprezentacji - O(E + V)
distance[V] = pocztkowo pusta lista najduszych odlegoci koczocych si w danych wierzchokach
V - liczba wierzchokw

longestPath() {
    var maxDistance = 0     // zmienna, w ktrej trzymamy dugo najduszej aktualnie cieki, warto kocowa bdzie rwna dugoci najduszej cieki w grafie
    var endOfLongestPath    // chcemy wiedzie, na ktrym wierzchoku koczy si najdusza cieka
    
    for vertex in topological_order {
        distance[vertex] = maxDistance(vertex, dist[])
        if distance[vertex] > maxDistance {
            maxDistance = dist[vertex]
            endOfLongestPath = vertex
        }
    }
    
    var path = getPath(endOfLongestPath, [endOfLongestPath])    // lista zawierajca nasz ciek
    reverse(path)    // zauwaamy, e dodawalimy wierzchoki od koca, zatem musimy odwrci kolejno
    for vertex in path {
        print(vertex)
    }
}

maxDistance(v) {
    if predecessors[v].count == 0 {    // zgodnie z zaoeniami jeli stopie wejciowy wierzchoka v jest rwny zeru to ustawiamy dist[v] = 0
        return 0
    }
    var maxDistance = 0
    for predecessor in predecessors[v] {
        if distance[predecessor] > maxDistance {
            maxDistance = distance[predecessor]    // szukamy maksymalnej dugoci wrd poprzednikw
        }
    }
    return maxDistance + 1
}

getPath(v, path[]) {
    var maxDistance = 0
    var previousVertex         // wierzchoek poprzednika v
    for neighbour in neighbours[v] {
        if distance[neighbour] > maxDistance {
            maxDistance = distance[neighbour]
            previousVertex = neighbour
        }
    }
    path += previousVertex
    if maxDistance == 0 {      // doszlimy do pocztku cieki
        return path
    } else {
        getPath(previousVertex, path)
    }
}

Zoono:

Zoono czasowa:

  • wykonanie sortowania topologicznego wykona si w czasie O(V + E)
  • w longestPath() przechodzimy przez kady wierzchoek grafu wywoujc na nim maxDistance(), ktry porusza si po wszystkich wierzchokach, ktre do niego trafiaj. Zatem poruszamy si po kadym wierzchoku i kadej krawdzi zatem wykona si to w czasie O(V + E).
  • getPath(v) porusza si po poprzednikach v, ktrych jest maksymalnie v - 1
  • zatem oglna zoono wynosi O(V + E)

Zoono pamiciowa:

  • graf podany w postaci listy ssiedztwa ma zoono pamiciow rwn O(V + E)
  • topological_order[] oraz distance[] maj zoono O(V)
  • path[] ma zoono co najwyej O(V - 1)
  • reszta zmiennych ma zoono O(1)
  • zatem oglna zoono wynosi O(V + E)
